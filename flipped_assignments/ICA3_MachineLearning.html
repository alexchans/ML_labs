Question 2 Is the code above a proper separation of training and testing sets for the given dataset (i.e., using KFold)? Why or why not?

The code above does not properly separate the training and testing sets for this binary classification problem because KFold does not preserve class proportions across folds. Given that the KDD Cup dataset is highly imbalanced, some folds may contain very few or no attack samples, leading to biased and unreliable evaluation results. A more appropriate choice is StratifiedKFold, which ensures that each fold maintains the same class distribution as the entire dataset, providing a fairer and more representative assessment of model performance.

Question 3: Is the evaluation metric chosen in the above code appropriate for the dataset (i.e., using accuracy)? Why or Why not?

No, accuracy is not appropriate because the dataset is highly imbalanced. A model could achieve high accuracy by mostly predicting the majority normal class while missing many attacks. Metrics like precision, recall, or F1-score better reflect performance on such imbalanced data.

Question 5: How would you change the method of cross validation to answer this question from DARPA? That is, how can you change your cross validation method to better mirror how your system will be used and deployed by DARPA when there are only 3-5 attack examples available for training? Note: you do not have access to these new training examples. You need to change you cross validation method with the existing data to answer this question.

To better mirror how the system will be used by DARPA when only 35 attack examples are available for training, we could implement a Few-Shot Simulation Strategy. In this approach, we modify our cross-validation to hold out entire subtypes of attacks, simulating unseen attack styles. For each held-out subtype, we include only 35 of its instances in the training set to represent the few new examples analysts might identify each day. The remaining instances of that subtype are used for testing, allowing us to measure how well the model generalizes to unseen variants based on minimal training data.

By rotating which attack subtypes are held out and averaging performance across folds, we can evaluate the models robustness under few-shot conditions. This method closely reflects DARPAs deployment environment, testing the models ability to adapt quickly and detect new attacks even when only a handful of examples are available for retraining.

def plot_filled(test_scores,train_x_axis, xlabel=''):
    
    test_mean = np.percentile(test_scores,50, axis=1)
    test_max = np.percentile(test_scores,95, axis=1) 
    test_min = np.percentile(test_scores,5, axis=1) 

    plt.plot(train_x_axis, test_mean,
             color='blue', linestyle='--',
             marker='s', markersize=5,
             label='validation set')

    plt.fill_between(train_x_axis,
                     test_min,
                     test_max,
                     alpha=0.15, color='blue')

    plt.grid(True)
    plt.xlabel(xlabel)
    plt.ylabel('Evaluation Criterion')
    plt.legend(loc='lower right')
    plt.tight_layout()

# compute mean and 95% CI using t = 2.26 for k = 10 folds
t_value = 2.26
def ci(scores):
    mean = np.mean(scores)
    se = np.std(scores, ddof=1) / np.sqrt(len(scores))
    return mean, mean - t_value*se, mean + t_value*se

mean1, low1, high1 = ci(evals1)
mean2, low2, high2 = ci(evals2)
mean3, low3, high3 = ci(evals3)

print(f"Model 1 (C=100): mean={mean1:.4f}, 95% CI=({low1:.4f}, {high1:.4f})")
print(f"Model 2 (C=1):   mean={mean2:.4f}, 95% CI=({low2:.4f}, {high2:.4f})")
